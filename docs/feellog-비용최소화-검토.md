# 필로그(feellog) 비용 최소화 개선 보고서

## 1. 한 줄 결론

**MVP 4개월의 현금 지출은 사실상 0원(도메인비 약 8,000원만)에 수렴 가능하나, 그것은 "절감안을 채택해서"가 아니라 "이미 계획서가 무료 티어·Phase2 분리·자체 자산을 정본으로 확정해 두었기 때문"이며, 진짜 레버는 현금이 아니라 1인 개발자의 개발-시간(약 4~6 개발월)과 시니어 신뢰선이다.**

**"MVP 현금 지출은 0원에 수렴 가능한가?" → 예, 단 조건부.** SaaS·인프라·스토어비·PG·이미지·분석 어디에도 신규 현금 절감은 거의 없다(전부 무료 티어이거나 이미 Phase2로 분리됨). 따라서 보고된 "수십~수백만원 절감" 수치는 대부분 **채택된 적 없는 외주/Pro 베이스라인을 깐 허수**다. 0원 수렴의 실질 조건은 ① Supabase Free 유지(+백업·일시정지 핑 대책), ② 스토어/PG/커뮤니티/지도 Phase2 분리 엄수, ③ 자체 일러스트·자체/PostHog 단일 분석, ④ **접근성·결제·개인정보·진단정확도 보호선 사수**다. 실제로 0원에 가까운 현금은 보장되지만, 그 대가로 백업 공백·일시정지·도달률 무신호 같은 비현금 리스크를 관리해야 한다.

## 4. 하지 말 것 (절감 허상 또는 시니어 제품 훼손)

**A. 절감 허상 — "효과 미미"로 솔직히 표기:**
- **BE-1 "4개월 138,000원 절감"** → 효과 미미. 본문 계획이 이미 무료시작을 명시(strawman 베이스라인). 실현 절감 ≈ 0.
- **BE-5 백업 6만원 절감** → ROI 음수. 1인 구현·복구리허설·키관리 시간 > 절감액.
- **BE-4 MVP egress/변환 절감** → 효과 미미~허상(수백원~0원). 효과는 모두 성장기 이후.
- **DATA-1/DATA-3 "외주 72만~192만 / 스톡 100만~300만 회피"** → 허수 앵커. 외주/스톡은 채택된 적 없는 3순위 대안. 진짜 효과는 공수뿐.
- **SCOPE-1 "1.5~2.5월·300~500만원"·SCOPE-7 "0.4~0.8월·외주 100~300만"·APP-5 "13만/년"·SAAS-4 "14만원 회피"·DATA-6 "15~20h"** → 모두 채택 안 된 베이스라인 기준 과장. 실질은 그보다 작거나(시간) 0(현금).
- **DATA-6 LLM 조기 태깅** → MVP 도입 금지(REJECT). 절감 허상 + 추천 정확도 게이트 훼손.
- **DATA-4 공공 시드 대량 적재** → 수익화 불가 카탈로그로 공급 리스크를 가리고 추천 균형을 왜곡.

**B. 시니어 제품/신뢰 훼손 — 절감 대상에서 절대 제외(PAY-6·SCOPE-8·DATA-7 보호선):**
- **접근성**: 본문 18px·글자크기 슬라이더(font_scale)·고대비 토글·48px 터치·WCAG AA 대비·스크린리더 레이블·alt_text 필수. 단, 3.5 체크리스트의 다수 항목은 Lighthouse CI로 **자동 보증되지 않으므로 수동 QA 병행** 필요(자동 가드 과대표현 주의).
- **결제/개인정보 신뢰**: 카드정보 미보관(PG 위임), 약관·동의·개인정보처리방침을 가입(S02)부터 포함, RLS ENABLE+FORCE, PII 최소수집·소프트삭제, 청약철회 14일·환불표·금액 서버검증·webhook 멱등·step-up 재인증(PG 도입 시 풀스코프). 이유: 섣부른 컷이 결제분쟁·법규위반·시니어 신뢰 붕괴라는 더 큰 비용을 부른다.
- **진단→5축→유형→매칭 정확도**: 채점/분류/매칭 수식 단위테스트 유지. 차별점 자체이며 컷 시 MVP 가설검증 무효.
- **BE-1 무백업 운영**: Free라도 pg_dump 백업 크론 없이 시니어 PII를 무백업으로 두지 말 것.
- **APP-5 데이터 없이 Android 고정**: iOS 시니어 배제 리스크.
- **DATA-2/3 활동카드에 AI 합성 사진**: 존재하지 않는 공방을 진짜처럼 보이는 허위표시 금지.

## 6. 비용 vs 리스크 — 한 줄 총평

**아껴야 할 것은 "현금"이 아니라 "1인 개발자의 시간"(스코프 격리·모노레포·OSS·분석 단일·EMA 축소로 약 2~4 개발월 회수)이고, 절대 깎지 말아야 할 것은 "시니어 신뢰선"(접근성·결제/개인정보·진단정확도)과 "데이터 안전"(Free라도 백업·일시정지 핑)이다 — 즉 보고된 절감액 대부분은 허수이니 숫자에 속지 말고, 진짜 레버인 시간을 아끼되 신뢰선은 0원이라도 사수하라.**
